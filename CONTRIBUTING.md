# Contributing to the AI-* Family

Thank you for your interest in contributing. We welcome improvements to workflow logic, templates, documentation, and bug fixes.

---

## Before You Start

### Contributor License Agreement (CLA)

All contributors must agree to our [Contributor License Agreement](./CLA.md) before their first contribution can be merged.

**What the CLA covers:**
- You grant a copyright and patent license for your contribution
- You represent that the work is your own (or you have rights to submit it)
- The licensor retains discretion over inclusion

**How to sign:** Include the following statement in your first Pull Request description:

> I have read and agree to the [Contributor License Agreement](./CLA.md).

## What You Can Contribute

| Area | Examples |
|------|----------|
| **Workflow logic** | Stage improvements, gate refinements, depth calibration |
| **Templates** | New output templates, template corrections |
| **Documentation** | README clarity, INSTALL instructions, typo fixes |
| **Bug fixes** | Broken cross-references, incorrect file paths, logic errors |
| **Platform support** | Installation instructions for additional AI coding assistants |

---

## What We Cannot Accept

- Project-specific content (packages must remain generic)
- Changes that remove or weaken the attribution requirement
- Modifications to LICENSE or NOTICE files without maintainer approval
- New packages (discuss in an issue first)

---

## How to Contribute

### 1. Open an Issue

For anything beyond a typo fix, open an issue first to discuss:
- What you want to change and why
- Which package(s) are affected
- Whether it's a bug fix, enhancement, or new feature

### 2. Fork and Branch

```bash
git fork
git checkout -b fix/your-description
```

### 3. Make Your Changes

- Follow existing file structure and naming conventions
- Keep templates generic (use `{placeholder}` syntax for project values)
- Maintain MUST/NEVER prescriptive language in steering files
- Ensure cross-references between files remain valid

### 4. Submit a Pull Request

- Reference the issue number
- Describe what changed and why
- Include the CLA agreement statement (first contribution only)
- Keep PRs focused — one concern per PR
