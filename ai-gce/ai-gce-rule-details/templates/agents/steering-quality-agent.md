# Steering Quality Agent

## Purpose

Analyzes the quality and health of `.kiro/steering/` files in the workspace. Checks context budget compliance (≤300 lines for always-inclusion files), prescriptive language quality, cross-file contradictions, freshness (stale rules), and ownership clarity. Prevents steering drift from degrading hook accuracy and AI behavior.

## When to Invoke

Call this agent **after editing any steering file** — to validate the edit didn't break anything.

- **Trigger:** Type `SQC__` in the chat prompt
- **Cadence:** After every steering file edit; also periodically (monthly health check)
- **Process point:** After saving changes to any `.kiro/steering/*.md` file

**Concrete examples:**
- "I just updated `tech-stack.md`" → call `SQC__` to validate
- "We're reorganizing steering files" → call `SQC__` after reorganization
- "Monthly steering health check" → call `SQC__` proactively
- "Hooks are firing incorrectly" → call `SQC__` — root cause may be stale steering

## Consequences of Skipping

**Immediate impact:**
- Edited steering may contradict other steering files
- Context budget exceeded → Kiro silently drops files from context
- Stale rules continue to be enforced (or worse, hooks pattern-match wrong paths)

**Accumulated debt (3+ missed checks):**
- Hooks fire on wrong file patterns (tech stack changed but `tech-stack.md` wasn't updated)
- AI behavior becomes unpredictable (contradictory steering = non-deterministic enforcement)
- Context window wasted on over-budget steering files
- Audit score drops in "steering governance" category

## Recovery

This agent is **self-healing** — it checks the CURRENT state of steering files regardless of when it's called.

1. Run `SQC__` now — it analyzes all steering files as they currently exist
2. Fix any violations found (contradictions, budget overages, stale rules)
3. If hooks were firing incorrectly due to stale steering:
   - Fix the steering file
   - Run AI-GCE re-derivation (Mode 2) to regenerate affected hooks
4. No retroactive work needed — steering quality is a point-in-time check

## Checks Performed

1. **Context budget:** No always-inclusion steering file exceeds 300 lines (Kiro context limit)
2. **Prescriptive language:** Rules use MUST/NEVER/ALWAYS (enforceable). Flagged: "should", "consider", "try to" (advisory, not enforceable)
3. **Cross-file contradictions:** No two steering files give conflicting instructions for the same concern (e.g., one says "use REST" while another says "use GraphQL" for the same endpoint)
4. **Freshness:** Are steering files referencing technologies, modules, or patterns that no longer exist in the codebase? (Cross-check against actual folder structure)
5. **Ownership clarity:** Does each steering file have a clear scope? (No two files governing the same domain without explicit layering)
6. **FileMatch accuracy:** Do conditional steering files (fileMatch) use correct glob patterns that match actual project paths?
7. **Provenance markers:** Do AI-GCE-generated steering files carry `<!-- generated by AI-GCE -->` markers? (Distinguishes them from AI-DWG output)

## Output

- **If all checks pass:** Brief summary — "Steering quality: ✅ {n} files checked, all healthy."
- **If violations found:**
  - Per-file report with violation type, line reference, and suggested fix
  - Compliance log entry appended to `compliance-log/events/{today-date}.jsonl`
  - Event type: `check`, hook: `steering-quality-agent`, result: `pass|fail|warn`

**Output location:** `.governance/compliance-log/events/`
