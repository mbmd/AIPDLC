## Hub Contribution Behavior

When AI-FLO renders this dashboard:

```
1. DETECT management_framework/dashboards/DASHBOARDS.md (hub marker).
2. IF hub exists:
   → CREATE or REFRESH flow-dashboard.md in dashboards/.
   → Update the AI-FLO row in DASHBOARDS.md "Contributing Dashboards" table.
3. IF no hub exists:
   → CREATE dashboards/ + DASHBOARDS.md + flow-dashboard.md.
4. NEVER modify another package's dashboard file.
```

## Refresh Trigger

The flow dashboard is refreshed:
- After every **routing event** (dispatch, advance, rework, cancel, conflict)
- After **periodic roll-up** (configurable cadence, default weekly)
- On **alert state change** (threshold breach, conflict detection)

## Scope Note

This is a **portfolio-level** dashboard. It only exists in portfolio mode (when AI-PPM/AI-FLO are
operating across multiple projects). In project mode (single project, end-to-end), AI-FLO does not
produce a dashboard — it produces the `roll-up-report.md` instead, which is consumed by AI-PPM.
