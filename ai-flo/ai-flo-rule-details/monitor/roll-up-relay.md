# Stage 9: Roll-Up Relay

## Purpose

Compile project positions, status, and health into a roll-up report for AI-PPM. This is the upward signal — giving the portfolio layer visibility into how its authorized projects are progressing.

---

## Trigger

This stage executes when:
- Operator requests: `roll-up` or `report`
- Configured schedule fires (e.g., every session start, weekly)
- AI-PPM explicitly requests status via its monitoring phase
- FLO detects a significant event that PPM should know about (escalation, cancellation)

---

## Steps

### Step 1: Gather Current State

From `flo-state.md`, extract per-project:

| Field | Source |
|-------|--------|
| Project ID | State entry |
| Project Name | State entry |
| Current Package | Position tracking (Stage 8) |
| Status | State entry (In Progress / Complete / Blocked / Cancelled) |
| Days in Stage | Calculate: today - last activity date |
| Priority | State entry |
| Blockers | Any active hold, conflict, or stall |
| Profile notes | Skipped/toggled packages |

### Step 2: Compile Portfolio Summary

```markdown
## Portfolio Summary

| # | Project ID | Name | Package | Status | Days | Priority | Health |
|---|------------|------|---------|--------|:----:|:--------:|:------:|
| 1 | PRJ-ERP-2026-001 | ERP Module | AI-DWG | In Progress | 3 | #1 | 🟢 |
| 2 | PRJ-CRM-2026-002 | CRM Rebuild | AI-ADLC | Stalled | 12 | #2 | 🔴 |
| 3 | PRJ-MOB-2026-003 | Mobile App | AI-PILC | In Progress | 2 | #3 | 🟢 |

Health key: 🟢 On track | 🟡 Warning (approaching threshold) | 🔴 Issue (stalled/blocked/conflict)
```

### Step 3: Compile Alerts Section

Any active issues that PPM should be aware of:

```markdown
## Active Alerts

| Project ID | Alert Type | Severity | Since | Detail |
|------------|-----------|:--------:|-------|--------|
| PRJ-CRM-2026-002 | Stall | Warning | 2026-06-06 | No movement at AI-ADLC for 12 days |
| PRJ-CRM-2026-002 | Conflict | Critical | 2026-06-15 | Priority collision (HOLDING) |
```

### Step 4: Compile Changes Since Last Roll-Up

```markdown
## Changes Since Last Report ({previous date})

| Date | Project ID | Event | From | To |
|------|------------|-------|------|----|
| 2026-06-14 | PRJ-ERP-2026-001 | Advanced | AI-POLC | AI-DWG |
| 2026-06-13 | PRJ-MOB-2026-003 | Dispatched | — | AI-PILC |
| 2026-06-15 | PRJ-CRM-2026-002 | Conflict detected | — | — |
```

### Step 5: Compile Metrics

```markdown
## Flow Metrics

| Metric | Value |
|--------|-------|
| Active projects | {N} |
| Projects on track (🟢) | {N} |
| Projects with warnings (🟡) | {N} |
| Projects with issues (🔴) | {N} |
| Avg days-in-stage (active) | {N} |
| Longest stall | {Project ID}: {N} days |
| Active conflicts (HOLDING) | {N} |
| Completions since last report | {N} |
| Cancellations since last report | {N} |
```

### Step 6: Produce Roll-Up Report

Write to `roll-up-reports/RU-{YYYY-MM-DD}.md`:

```markdown
# Roll-Up Report: {date}

**Generated by:** AI-FLO v1.0.0
**Topology:** Mode {X}
**Projects tracked:** {N}
**Period:** {since last report date} → {today}

{Portfolio Summary table}
{Active Alerts table}
{Changes Since Last table}
{Flow Metrics table}

---

## Recommendations

{If any issues exist, FLO suggests actions:}
- PRJ-CRM-2026-002: Stalled 12 days. Recommend: check with architecture team or escalate.
- {etc.}

---

*This report is consumable by AI-PPM's monitoring phase for portfolio-level aggregation.*
```

### Step 7: Relay Decision

```
📊 Roll-Up Report Ready: RU-{date}.md

  Projects: {N} tracked | 🟢 {N} | 🟡 {N} | 🔴 {N}
  Changes: {N} events since last report
  Alerts: {N} active

  [V] View full report | [S] Send to PPM folder | [B] Both
```

If Mode 2/3 and PPM is in this workspace: place report where AI-PPM expects it.
If Mode 1: report is already accessible (same workspace).

---

## Gate

Operator confirms roll-up content before it's "sent" (placed in PPM's intake area). FLO doesn't auto-publish without confirmation (v1.0 advisory model).

---

## Logging

- Log: routing-log entry Type=Roll-Up, Notes="Report RU-{date} generated, {N} projects"
- No spine entry for routine roll-ups (they're operational, not decisions)
- Exception: if roll-up contains an ESCALATED issue → spine entry already exists from Stage 10
