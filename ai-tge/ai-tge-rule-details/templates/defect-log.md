## Status Lifecycle Reference

```
Open → Investigating → Fixed → Verified → Closed
                 ↓                    ↓
            (Won't Fix)          (Reopened → Open)
```

| Transition | Trigger | Who |
|-----------|---------|-----|
| Open → Investigating | Developer begins analysis | Developer |
| Investigating → Fixed | Code fix applied, PR merged | Developer |
| Fixed → Verified | QA confirms fix works, no regression | QA |
| Verified → Closed | Regression prevention confirmed | QA/PM |
| Any → Won't Fix | Accepted risk, documented rationale | PM/Architect |
| Fixed/Verified → Reopened | Fix didn't work or defect recurred | QA |
