## The Problem

Organizations don't fail one project at a time — they fail portfolios. Too many projects fighting for the same people. Strategic initiatives starved while "keep the lights on" work consumes everything. No visibility across the portfolio until quarterly steering committees where it's too late to act.

The typical pattern: each project is initiated, prioritized, and tracked independently. The PMO maintains a spreadsheet called "Portfolio View" that's updated monthly (if anyone remembers). Resource conflicts are discovered in retrospective, not in planning. Strategic alignment is a slide in a deck, not an enforced constraint.

The result: organizations do too much, finish too little, and can't explain why the portfolio mix doesn't match the strategy they approved.

---

## The Solution

AI-PPM provides continuous portfolio governance — intake, prioritization, authorization, monitoring, and rebalancing — as a single adaptive engine. It reads project data from across the AI-* Family (via AI-FLO) and provides the cross-project view that individual project packages cannot.

**One portfolio. All projects visible. Decisions auditable. Priorities enforced.**

The AI acts as a Portfolio Analyst / PMO Advisor. It doesn't run individual projects — it governs the collection: which projects should start, which should pause, which are dragging the portfolio down, and whether the portfolio mix still serves the strategy.

---

## How It Works

```
PIPs + Idea Briefs → INTAKE → PRIORITIZE → AUTHORIZE → MONITOR → OPTIMIZE → Portfolio Register
```

**5 Phases, 10 Stages, continuous:**

| Phase | What Happens | Key Output |
|-------|-------------|------------|
| Intake | Register projects from PIPs/Idea Briefs into the portfolio | Portfolio Register entry |
| Prioritization | Align to strategy, rank projects against each other | Strategic Alignment Map, Prioritization Scorecard |
| Authorization | Governance gate: admit, pause, resume, or retire | Governance Decision Record, Dispatch Authorization |
| Monitoring | Ingest cross-project data via AI-FLO; surface patterns | Portfolio Health Dashboards |
| Optimization | Rebalance priorities based on actuals; retire failing projects | Rebalancing Proposal, Retirement Record |

AI-PPM is **continuous** — not a one-pass lifecycle. The portfolio is alive. New projects arrive, running projects report health, underperformers get reviewed, and the portfolio mix is rebalanced as reality diverges from plans.

## Key Differentiators

### 1. Cross-Layer Architecture (FLO-Mediated)

AI-PPM lives in the Portfolio layer. It never reaches into individual projects — it receives data via AI-FLO (the layer-boundary courier). This keeps project autonomy intact while giving portfolio visibility. Same-layer siblings (AI-ILC, AI-PILC) are read directly.

### 2. Governance Gates, Not Just Views

Most portfolio tools are dashboards — they show data but don't enforce decisions. AI-PPM has a governance gate (Stage 5) where projects are formally admitted, paused, or retired. Every decision produces an auditable record with rationale.

### 3. 7 Opt-In Extensions

Core covers 85% of portfolio management. Extensions handle advanced patterns:
- Portfolio Balancing & Visualization
- What-If Scenario Modeling
- Cross-Project Dependency Mapping
- Capacity & Demand Management
- Investment Themes / Strategic Buckets
- Financial Governance
- Benefits Realization Aggregation

### 4. Descoped Duplication

AI-PPM doesn't reinvent per-project metrics. It reads them from where they already live (AI-PILC budget, AI-POLC backlog health, AI-GCE compliance, AI-TGE test coverage). No double-entry. No conflicting numbers.

---

## Position in AIFLC — the AI-* PDLC Family

AI-PPM is the **top of the Portfolio layer** — the portfolio governor:
- Reads from **AI-PILC** (PIPs) and **AI-ILC** (Idea Briefs) — same layer, direct
- Receives from **Project layer** (health, progress, financials) — cross-layer, via AI-FLO
- Dispatches to **Project layer** (authorization to proceed) — cross-layer, via AI-FLO

It is the only package with a portfolio-wide view. All other packages operate on one project at a time.
